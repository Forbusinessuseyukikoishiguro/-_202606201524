# うさうさプロンプター 🐰 テストの使い方

`usausa-prompter.test.mjs` は、ツール本体に組み込まれたテスト用API
（`window.__usausa`）を jsdom 経由で検証する、単一ファイルの商用テストです。

## 必要なもの
- Node.js（v18 以上）
- jsdom

## 準備（jsdom の導入）
プロジェクト用フォルダで（テスト対象HTMLと同じ場所が手軽です）：

```bash
npm init -y
npm i jsdom
```

> グローバル導入（`npm i -g jsdom`）でも動きますが、ESM では
> `NODE_PATH=$(npm root -g) node ...` のように `NODE_PATH` の指定が要ります。
> 迷ったら上のローカル導入が確実です。

## 実行
テスト対象HTMLを**同じフォルダ**に置けば、引数なしで自動検出します
（`usausa-prompter*.html` を探し、PRO版を優先します）。

```bash
node usausa-prompter.test.mjs
```

対象HTMLを明示したいときは引数で渡します：

```bash
node usausa-prompter.test.mjs ./usausa-prompter-PRO-260620-062355.html
```

環境変数でも指定できます：

```bash
USAUSA_HTML=./usausa-prompter.html node usausa-prompter.test.mjs
```

## 終了コード
- すべて成功 … `0`
- 1件でも失敗 … `1`（CI でそのまま判定に使えます）

失敗時は末尾に「失敗した項目」の一覧が出ます。

## テストの内訳（計 131 項目・6カテゴリ）
1. **単体** … 純粋関数（文字数・範囲クランプ・エスケープ・コントラスト比/WCAG・
   応援メッセージ・CSV整形・行/ページ解析）
2. **防御（サニタイズ）** … メモ・履歴・実績ログ・台本スロットの不正値除去
3. **集計** … 読み上げ実績の平均・差分（runSummary）
4. **結合（直列化の往復）** … serialize/deserialize（スキーマ v12）、
   全フィールドの保持、50件上限、既定値、壊れた入力への耐性
5. **DOM・刻印** … 主要ボタン/モーダルの存在、追加機能のUI
   （実績・コントラスト・台本2本・監督/応援レビュー・カウント音・鈴・てんとう虫）、
   表紙（ON AIR・応援テロップ・うさぎ写真）、バージョン/「テスト済み」刻印、夜空配色
6. **モンキー** … 異常入力でクラッシュしないこと

## メモ
- AI機能（相談・監督/応援レビュー・整形）はオンライン専用のため、
  ネットワーク通信は行わず、ボタンやUIの存在のみを検証します。
- 画像の見た目やスクロールの実描画は jsdom では描画されないため、
  本テストはロジックとDOM構造を対象にしています（見た目は実機でご確認ください）。
